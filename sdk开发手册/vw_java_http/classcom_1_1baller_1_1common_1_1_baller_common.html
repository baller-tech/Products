详细描述

login 和 logout 是 baller sdk 的公共接口，与使用 baller sdk 的进程相关。不同能力的 baller sdk 使用相同的 login 和 logout。 同一个进程使用多种能力的 baller sdk 时，只需调用一次 login 和 logout 即可。
比如某一个进程既使用了 baller asr sdk 又使用了 baller tts sdk，只需在调用 baller asr sdk 和 baller tts sdk 的接口之前调用一次 login，在 baller asr sdk 和 baller tts sdk 所有session 都回收完成之后调用一次 logout即可。

警告
授权中SDK端并发数限制的说明:
SDK端并发数限制是指可以调用BallerXXX.sessionBegin创建session的数量，是对一台机器中所有进程可创建session数量总和的限制。
SDK端并发数是一台机器中的全局限制，与login和与logout的调用无关；必须显式的调用BallerXXX.sessionEnd接口，才会将已使用的并发数减一；当应用程序的进程退出（正常或异常退出）时，必须对已创建的session调用BallerXXX.sessionEnd接口，否则已使用的并发数不会减少。
如果授权信息中有SDK端并发数限制，建议应用程序添加异常处理逻辑，当应用程序的进程出现异常需要结束时，对已创建的session调用BallerXXX.sessionEnd接口后再结束。否则已使用的并发数永远不会减少，直到机器重启。
