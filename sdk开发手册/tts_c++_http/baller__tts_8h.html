调用 baller tts sdk 的接口前需先调用 BallerLogin 进行登录和调用者身份的验证。
BallerLogin 接口调用成功后需调用 BallerTTSSessionBegin 接口创建一个语音合成功能的session，语音合成操作都是基于session的，当不再需要语音合成功能后，需调用 BallerTTSSessionEnd 接口回收该 session。
BallerTTSSessionBegin 和 BallerTTSSessionEnd 需成对出现，如果缺少 BallerTTSSessionEnd 的调用，会导致SDK的端并发数不会减少。
如果授权信息中有SDK端并发数限制，建议应用程序添加异常处理逻辑，当应用程序的进程出现异常需要结束时，对已创建的session调用BallerTTSSessionEnd接口后再结束。否则已使用的并发数永远不会减少，直到机器重启。

在文件 baller_tts.h 中定义.

◆ BallerTTSGet()

BALLER_API int BallerTTSGet ( const baller_session_id  session_id,
char **  out_data,
int *  out_len 
)

获取语音合成的结果

参数
session_id处理语音合成的 session 的 session id。
out_data合成结果的数据。
out_len合成结果的数据长度。
返回
0表示成功,获取结果结束；14004(BALLER_MORE_RESULT)表示还有未取完的合成结果，需继续调用BallerTTSGet接口；其他表示失败，详情见错误码。
警告
调用完BallerTTSPut接口后，需要调用BallerTTSGet接口获取合成结果。
如果返回值为0(BALLER_SUCCESS)表示合成结果已全部获取完毕，不需继续调用BallerTTSGet接口；如果返回值为14004(BALLER_MORE_RESULT)表示还有未取完的数据，需继续调用BallerTTSGet接口；返回其他表示出错，不需要继续调用。
*out_data的内存管理由SDK控制，在第n（n>1）次调用BallerTTSGet接口时，SDK会释放n-1（n>1）次调用时分配的内存空间，不需要调用者释放。若调用者需要暂存之前n-1次获取的识别结果，需调用者分配空间保存。
注解
while(1)
{
int ret = BallerTTSGet(session_id, out_data, out_len);
if (BALLER_MORE_RESULT == ret)
{
if (*out_data && out_len > 0)
{
// 处理获取到的结果
}
continue;
}
else if (BALLER_SUCCESS == ret)
{
if (*out_data && out_len > 0)
{
// 处理获取到的结果
}
break;
}
else
{
printf("BallerTTSGet failed. error code: %d\n", ret);
break;
}
}

向 session中传递文本数据

参数
session_id语音合成session 的 session id。
params处理文本数据的参数，详见下表：
参数 参数介绍 是否必填 可设置的值 值介绍
input_mode 输入模式 否 once 【默认值】将所有文本数据一次性传入
continue 【暂不支持】文本数据时实时传入且不是最后一次传入
end 【暂不支持】文本数据实时传入且本次是最后一次传入
in_data文本数据。
返回
0表示成功；其他表示失败，详情见错误码。
注解
const char* params = "input_mode=once";
const char* text = "今天天气不错";
int ret = BallerTTSPut(session_id, params, text);
if (BALLER_SUCCESS != ret)
{
printf("BallerTTSPut failed. error code: %d\n", ret);
}

参数 参数介绍 是否必填 可设置的值 值介绍
res_dir 资源文件所在的路径 是 资源文件所在的路径 资源文件所在的路径

iii 彝语

gpu 【暂不支持】在GPU上进行计算

0表示成功；其他表示失败，详情见错误码。

const char* session_prams = "res_dir=data,language=zho,engine_type=local,hardware=cpu_slow,sample_rate=16000,sample_size=16";
int ret = BallerTTSSessionBegin(session_prams, &session_id);
if (BALLER_SUCCESS != ret)
{
printf("BallerTTSSessionBegin failed. error code: %d\n", ret);
}

◆ BallerTTSWorkingThread()

BALLER_API int BallerTTSWorkingThread ( int  num)

设置语音合成session使用的逻辑cpu个数

参数
session_id语音合成session 的 session id。
num使用的逻辑cpu的个数（建议值为4或者8）
返回
0表示成功；其他表示失败，详情见错误码。
警告
BallerTTSWorkingThread 接口需在每个调用 BallerTTSPut 的线程中调用一次（仅需调用一次），且必须早于 BallerTTSPut 调用。
BallerTTSWorkingThread 接口参数num会影响语音合成的效率，建议值为4或者8，与cpu型号有关，调用者可以在自己的目标环境中进行测试，找到一个性能最好的值。
注解
int ret = BallerTTSWorkingThread(4);
if (BALLER_SUCCESS != ret)
{
printf("BallerTTSWorkingThread failed. error code: %d\n", ret);
}
