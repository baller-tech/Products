详细描述

login 和 logout 是 baller sdk 的公共接口，与使用 baller sdk 的进程相关。不同能力的 baller sdk 使用相同的 login 和 logout。 同一个进程使用多种能力的 baller sdk 时，只需调用一次 login 和 logout 即可。
比如某一个进程既使用了 baller asr sdk 又使用了 baller tts sdk，只需在调用 baller asr sdk 和 baller tts sdk 的接口之前调用一次 login，在 baller asr sdk 和 baller tts sdk 所有session 都回收完成之后调用一次 logout即可。

使用baller sdk时必须需取得READ_PHONE_STATE和INTERNET授权；如果授权文件/资源文件保存在sd卡中，需取得READ_EXTERNAL_STORAGE授权；如果需要向sd卡中写日志，需取得WRITE_EXTERNAL_STORAGE授权。

警告
授权中SDK端调用间隔限制的说明:
SDK端调用间隔限制是指两次完整的BallerXXXPut接口调用的间隔限制，以能力类型为最小粒度。
当出现以下两种情况之一时才算一次完整的BallerXXXPut调用：
  • 调用BallerXXXPut接口，当input_mode为once时算一次完整的调用。
  • 调用BallerXXXPut接口，当input_mode为continue或end时，仅当input_mode变为end时，才算一次完整的调用。同一个session中，当连续多次的input_mode都为continue时，不算一次完整的调用，也不会受SDK端并发数的限制。
