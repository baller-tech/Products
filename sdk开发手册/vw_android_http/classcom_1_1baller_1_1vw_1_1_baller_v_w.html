详细描述

调用 baller vw sdk 的接口前需先调用 login 进行登录和调用者身份的验证。
login 接口调用成功后需调用 sessionBegin 接口创建一个语音唤醒功能的session，语音唤醒操作都是基于session的，当不再需要语音唤醒功能后，需调用 sessionEnd 接口回收该 session。

put 接口用于将音频数据传递给sdk，支持两种传递模式：

  • 将音频数据一次全部传入sdk，适用于对已经录制好的音频文件进行唤醒检测
  • 将音频数据实时传递给sdk，适用于录音设备一边采集音频，一边将采集到的音频提供给SDK

native int com.baller.vw.BallerVW.abort ( )

终止已传入音频数据的唤醒词检测

将音频数据实时传入put接口时，如果已经发现唤醒词且不需要继续等待唤醒时需调用该接口，结束本次唤醒操作。 SDK会丢弃已传入put接口但未来得及处理的音频数据。调用完该接口之后，如果继续调用put接口，会重新开始唤醒。

返回
0表示成功；其他表示失败，详情见错误码。
注解

// 调用sessionBegin接口，创建session
...
// 调用put接口，传入音频数据
...

int ret = wakeInstance.abort();
if (BallerErrorCode.BALLER_SUCCESS != iRet)
{
System.out.printf("call abort falied %d\n", iRet);
}

参数 参数介绍 是否必填 可设置的值 值介绍
wakeup_word 唤醒词 是 使用英文分号分割不同的唤醒词；使用空格分割唤醒词中不同的字/单词
中文使用大写拼音表示；英文使用小写字母 		唤醒词

8 【暂不支持】音频采用8KHz的采样率进行采样
sample_size 音频的每个采样点大小 否 16 【默认值】音频每个采样点16bit
8 【暂不支持】音频每个采样点8bit
[in]dataDir唤醒资源文件的路径，不需要最后的'/'分隔符。如果为assets中的路径则assetManager参数必须传有效的AssetManager实例；如果为sdcard中的路径assetManager参数必须传null
[in]assetManagerandroid.content.res.AssetManager实例
