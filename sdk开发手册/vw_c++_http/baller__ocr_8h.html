调用 baller ocr sdk 的接口前需先调用 BallerLogin 进行登录和调用者身份的验证。
BallerLogin 接口调用成功后需调用 BallerOCRSessionBegin 接口创建一个图像识别功能的session，图像识别操作都是基于session的，当不再需要图像识别功能后，需调用 BallerOCRSessionEnd 接口回收该 session。
BallerOCRSessionBegin 和 BallerOCRSessionEnd 需成对出现，如果缺少 BallerOCRSessionEnd 的调用，会导致SDK的端并发数不会减少。
如果授权信息中有SDK端并发数限制，建议应用程序添加异常处理逻辑，当应用程序的进程出现异常需要结束时，对已创建的session调用BallerOCRSessionEnd接口后再结束。否则已使用的并发数永远不会减少，直到机器重启。

◆ BallerOCRGet()

BALLER_API int BallerOCRGet ( const baller_session_id  session_id,
char **  out_data,
int *  out_len 
)

获取图像识别结果

参数
session_id处理图像识别的 session 的 session id。
out_data识别结果的数据。
out_len识别结果的数据长度。
返回
0表示成功,获取结果结束；14004(BALLER_MORE_RESULT)表示还有未取完的识别结果，需继续调用BallerOCRGet接口；其他表示失败，详情见错误码。
警告
调用完BallerOCRPut接口后，需要调用BallerOCRGet接口获取识别结果。
如果返回值为0表示识别结果已全部获取完毕，不需继续调用BallerOCRGet接口；如果返回值为14004(BALLER_MORE_RESULT)表示还有未取完的数据，需继续调用BallerOCRGet接口；返回其他表示出错，不需要继续调用。
*out_data的内存管理由SDK控制，在第n（n>1）次调用BallerOCRGet接口时，SDK会释放n-1（n>1）次调用时分配的内存空间，不需要调用者释放。若调用者需要暂存之前n-1次获取的识别结果，需调用者分配空间保存。
注解
while(1)
{
int ret = BallerOCRGet(session_id, out_data, out_len);
if (BALLER_MORE_RESULT == ret)
{
if (*out_data && out_len > 0)
{
// 处理获取到的结果
}
continue;
}
else if (BALLER_SUCCESS == ret)
{
if (*out_data && out_len > 0)
{
// 处理获取到的结果
}
break;
}
else
{
printf("BallerOCRGet failed. error code: %d\n", ret);
break;
}
}

◆ BallerOCRPut()

BALLER_API int BallerOCRPut ( const baller_session_id  session_id,
const char *  params,
const void *  in_data,
const int  in_len 
)

向 session 中传入图像数据

参数
session_id处理图像识别的 session 的 session id。
params处理图像数据的参数，详见下表：
参数 参数介绍 是否必填 可设置的值 值介绍
input_mode 输入模式 否 once 【默认值】将图像数据一次性传入
continue 【暂不支持】图像数据分批传入且不是最后一次传入
end 【暂不支持】图像数据分批传入且是最后一次传入
in_data图像数据。
in_len采样数据的长度；以字节单位。
返回
0表示成功；其他表示失败，详情见错误码。
备注
目前支持处理格式为 jpg, jpeg, bmp, png, gif, tif, tiff 的图像数据。
注解
char* img_data = 0;
int img_len = 0;
// 获取图像信息，将获取的图像信息保存到img_data变量中，将获取的图像信息长度保存到img_len中
const char* params = "input_mode=once";
int ret = BallerOCRPut(session_id, params, (void*)img_data, img_len);
if (BALLER_SUCCESS != ret)
{
printf("BallerOCRPut failed. error code: %d\n", ret);
}

◆ BallerOCRSessionBegin()

BALLER_API int BallerOCRSessionBegin ( const char *  session_params,
baller_session_id *  session_id 
)

开始一个图像识别 sessioin

参数
[in]session_params启动session的参数，详见下表：
参数 参数介绍 是否必填 可设置的值 值介绍
res_dir 资源文件所在的路径 是 资源文件所在的路径 资源文件所在的路径
engine_type 引擎类型 否 local 【默认值】使用本地引擎
cloud 【暂不支持】使用云端引擎
hardware 用来计算的硬件信息；当engine_type=local时有效 否 cpu_slow 【默认值】在CPU上进行计算
cpu_fast 【暂不支持】在CPU上快速进行计算
gpu 【暂不支持】在GPU上进行计算
mode 图像识别模式 否 multi_row 【默认值】多行识别
char 【暂不支持】单字符识别
row 【暂不支持】单行识别
column 【暂不支持】单列识别
multi_column 【暂不支持】多列识别
[out]session_id本次启动的session id
返回
0表示成功；其他表示失败，详情见错误码。
注解
baller_session_id session_id = BALLER_INVALID_SESSION_ID;
const char* session_prams = "res_dir=data,engine_type=local,hardware=cpu_slow,mode=multi_row";
int ret = BallerOCRSessionBegin(session_prams, &session_id);
if (BALLER_SUCCESS != ret)
{
printf("BallerOCRSessionBegin failed. error code: %d\n", ret);
}

版本获取接口

获取OCR SDK的版本信息。

参数
[out]version版本信息的字符数组，需调用者分配内存空间。
[in]lenversion参数内存空间的字节数。
返回
无。
备注

注解
char version[128];
memset(version, 0, 128);
BallerOCRVersion(version, 128);
printf("baller sdk version: %s\n", version);
