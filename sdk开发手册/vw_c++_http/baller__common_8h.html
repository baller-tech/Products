BallerLogin 和 BallerLogout 是 baller sdk 的公共接口，与使用 baller sdk 的进程相关。不同能力的 baller sdk 使用相同的 BallerLogin 和 BallerLogout。 同一个进程使用多种能力的 baller sdk 时，只需调用一次 BallerLogin 和 BallerLogout 即可。
比如某一个进程既使用了 baller asr sdk 又使用了 baller tts sdk，只需在调用 baller asr sdk 和 baller tts sdk 的接口之前调用一次 BallerLogin，在 baller asr sdk 和 baller tts sdk 所有session 都回收完成之后调用一次 BallerLogout即可。

在文件 baller_common.h 中定义.

BALLER_API int BallerLogin ( const char *  login_params)

baller sdk 登录接口

baller sdk 用户身份确认并进行全局初始化的操作。调用baller sdk的其他接口前需要先调用该接口。

参数
[in]login_params登录时使用的参数，详见下表：
参数 参数介绍 是否必填 可设置的值 值介绍
org_id 客户ID 是 由北京大牛儿科技发展有限公司统一分配 由北京大牛儿科技发展有限公司统一分配
app_id 应用ID 是 由北京大牛儿科技发展有限公司统一分配 由北京大牛儿科技发展有限公司统一分配
app_key 应用KEY 是 由北京大牛儿科技发展有限公司统一分配 由北京大牛儿科技发展有限公司统一分配
license 授权文件所在的路径和名称 是 授权文件所在的路径和名称 授权文件所在的路径和名称
log_level 日志级别
不设置则不会输出日志 		否 debug 显示debug级别的日志
info 显示info级别的日志
warning 显示warning级别的日志
error 显示error级别的日志
log_path 日志保存路径 否 日志文件的输出路径 日志文件的输出路径
返回
0表示成功；其他表示失败，详情见错误码 。
警告
授权中SDK端并发数限制的说明:
SDK端并发数限制是指可以调用BallerXXXSessionBegin创建session的数量，是对一台机器中所有进程可创建session数量总和的限制。
SDK端并发数是一台机器中的全局限制，与BallerLogin和与BallerLogout的调用无关；必须显式的调用BallerXXXSessionEnd接口，才会将已使用的并发数减一；当应用程序的进程退出（正常或异常退出）时，必须对已创建的session调用BallerXXXSessionEnd接口，否则已使用的并发数不会减少。
如果授权信息中有SDK端并发数限制，建议应用程序添加异常处理逻辑，当应用程序的进程出现异常需要结束时，对已创建的session调用BallerXXXSessionEnd接口后再结束。否则已使用的并发数永远不会减少，直到机器重启。
注解
const char* login_params = "org_id=1157870520697357744,app_id=1157889509295128642,app_key=f497dae1bb3a5c2490b2f97dcbcc579c,license=./baller_sdk.license,log_level=debug,log_path=./baller_log/";
int ret = BallerLogin(login_params);
if (BALLER_SUCCESS ！= ret)
{
printf("BallerLogin failed. error code: %d\n", ret);
}
